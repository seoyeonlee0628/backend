# 진단 결과: 연간 최대 수혜액 · 가장 빠른 마감 계산 방식

## 개요

`POST /predict` 응답의 `amount`(연간 최대 수혜액)와 `deadline`(가장 빠른 마감)은
추천된 서비스 목록을 기반으로 자동 계산된다.

---

## 연간 최대 수혜액 (`amount`)

### 계산 흐름

```
각 서비스의 amt 문자열
  → parse_amount_to_annual() 로 연간 만원 단위 정수 변환
  → 전체 합산
  → format_annual_amount() 로 문자열 포맷
```

### `parse_amount_to_annual(amt_str)` 변환 규칙

| 입력 예시 | 변환 결과 | 설명 |
|-----------|-----------|------|
| `"월 50만원"` | 600만원/년 | 월 → ×12 |
| `"최대 300만원"` | 300만원/년 | 총액 그대로 |
| `"3년 최대 1,440만원"` | 480만원/년 | 총액 ÷ 기간(년) |
| `"5년 최대 5,000만원"` | 1,000만원/년 | 총액 ÷ 기간(년) |
| `"확인 필요"` | 0 (합산 제외) | 파싱 불가 |

> 서비스 요약문(`summary`)에서 `extract_amount()`가 먼저 금액 문자열을 뽑고,
> `parse_amount_to_annual()`이 연간 값으로 환산한다.

### `format_annual_amount(total_man)` 출력 형식

| 합산 값 | 출력 예시 |
|---------|-----------|
| 0 이하 | `"복지로에서 확인 필요"` |
| 1,200만원 | `"연간 최대 1,200만원"` |
| 10,000만원 (1억) | `"연간 최대 1억원"` |
| 12,400만원 | `"연간 최대 1억 2,400만원"` |

## 가장 빠른 마감 (`deadline`)

### 계산 흐름

```
각 서비스의 deadline 문자열 (extract_deadline() 결과, ISO 형식 YYYY-MM-DD)
  → 오늘 이후 날짜만 수집
  → 최솟값(가장 빠른 날짜) 선택
  → "M월 D일까지" 형식으로 포맷
  → 해당 날짜 없으면 "상시" 반환
```

### `find_earliest_deadline(services)` 처리 규칙

- `"상시"`, `"정보 없음"`, `"확인 필요"` → 건너뜀
- `YYYY-MM-DD` 형식만 파싱 (다른 형식은 건너뜀)
- **오늘 이전 날짜는 무시** (이미 지난 마감)
- 유효한 날짜가 하나도 없으면 `"상시"` 반환

### 출력 예시

| 서비스 deadline 목록 | 결과 |
|----------------------|------|
| `["상시", "상시", "상시"]` | `"상시"` |
| `["2026-08-31", "상시", "2026-12-31"]` | `"8월 31일까지"` |
| `["2025-01-01", "2026-09-30"]` (2025-01-01은 이미 지남) | `"9월 30일까지"` |

## 알려진 금액 사전 (`KNOWN_AMOUNTS`)

지자체 API 요약문에 금액 정보가 없는 경우를 보완하기 위해 주요 정책 금액을 하드코딩.
서비스 이름에 키워드가 포함되면 요약문 파싱보다 우선 적용된다.

| 서비스 키워드 | 금액 |
|---------------|------|
| 구직촉진수당 / 국민취업지원제도 | 월 50만원 |
| 청년내일저축계좌 | 3년 최대 1,440만원 (→ 연 480만원) |
| 청년도약계좌 | 5년 최대 5,000만원 (→ 연 1,000만원) |
| 청년월세 | 월 20만원 (→ 연 240만원) |
| 주거급여 | 월 최대 32만원 (→ 연 384만원) |
| 내일배움카드 / 국민내일배움카드 | 최대 500만원 |
| 청년내일채움공제 | 2년 최대 1,200만원 (→ 연 600만원) |

> 새로운 정책이 추가될 때 `KNOWN_AMOUNTS` 딕셔너리에 키워드·금액을 추가하면 된다.

## 주의사항

- `extract_amount()`는 서비스 요약문에서 첫 번째 패턴만 추출한다.
  실제 지원액이 요약문에 없으면 `"확인 필요"`가 반환되어 합산에서 제외된다.
- 마감일은 복지로 요약문 파싱 결과이므로, 요약문에 날짜가 없으면 `"상시"`로 처리된다.
- 연간 수혜액은 **수령 가능한 이론적 최댓값**이며, 실제 수령액은 자격 심사 결과에 따라 다를 수 있다.
